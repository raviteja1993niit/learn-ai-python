# ✅ Great Expectations — Data Quality

## What is Great Expectations?
Great Expectations (GX) is an open-source Python library for validating, documenting, and profiling
your data. You define human-readable "expectations" about your data, then GX validates datasets
against them and generates rich HTML data docs — turning data quality checks into living documentation.

## Why Learn It?
- Catch data quality regressions before they silently corrupt ML model training
- Auto-generates beautiful HTML data docs: data dictionary + validation results in one place
- 50+ built-in expectations covering nulls, ranges, uniqueness, regex patterns, and schema
- Integrates natively with Airflow, Prefect, dbt, and Databricks as a validation gate
- Supports Pandas, Spark, SQL databases, and cloud data warehouses via the same API
- More expressive and ML-focused than Pydantic/Pandera for dataset-level validation

## Key Concepts
```python
import great_expectations as gx
import pandas as pd

# Initialize a Data Context (project config + store)
context = gx.get_context()

# Connect to a Pandas DataFrame as a data source
df = pd.read_csv("data/train.csv")
validator = context.sources.pandas_default.read_dataframe(df)

# Core expectations — column-level constraints
validator.expect_column_to_exist("age")
validator.expect_column_values_to_not_be_null("age")
validator.expect_column_values_to_be_between("age", min_value=0, max_value=120)
validator.expect_column_values_to_be_in_set("gender", value_set=["M", "F", "Other"])
validator.expect_column_values_to_match_regex("email", r"^[\w.+-]+@[\w-]+\.[a-z]{2,}$")
validator.expect_column_to_exist("label")
validator.expect_column_unique_value_count_to_be_between("label", min_value=2, max_value=10)

# Save the expectation suite
validator.save_expectation_suite(discard_failed_expectations=False)
```

```python
# Checkpoint — run validation and build data docs
import great_expectations as gx

context = gx.get_context()

checkpoint = context.add_or_update_checkpoint(
    name="daily_training_data_check",
    validations=[
        {
            "batch_request": batch_request,
            "expectation_suite_name": "train_data.critical",
        }
    ],
)

result = checkpoint.run()
context.build_data_docs()   # generates HTML report in gx/uncommitted/data_docs/

# Raise an error if any expectation fails (use as a pipeline gate)
if not result.success:
    raise ValueError("Data quality check failed — aborting training pipeline")
```

    """Custom: mean of the column must be > 0."""

    _validators = []

        super().validate_configuration(config)

## Learning Path
1. `pip install great_expectations` and run `great_expectations init` in your project
2. Connect a Pandas CSV as a data source and run `great_expectations suite new`
3. Add 5 core expectations (null, between, in_set, regex, exists) and save the suite
4. Create a `Checkpoint` and run it; open the generated HTML data docs in your browser
5. Add a failing expectation and verify the pipeline gate (`raise ValueError`) fires
6. Write one custom `ColumnExpectation` subclass with a business-specific rule
7. Integrate GX as a task in a Prefect or Airflow flow before the training step
8. Compare GX suites with Pandera schemas and Pydantic models for the same dataset
