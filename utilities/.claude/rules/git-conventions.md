# Git Conventions

## Branch Naming

```
feature/<TICKET-ID>_<SHORT_DESCRIPTION>
fix/<TICKET-ID>-<SHORT_DESCRIPTION>
```

Examples:
- `feature/G1198_18131_CHASE_TSPI_MASKING_LOGS`
- `fix/G1198-17891-null-pointer-in-mapper`

## Commit Message Format

```
<TICKET-ID>: <Short imperative description>
```

Examples:
- `G1198-16490: Update circuit breaker threshold configuration`
- `PROJ-1234: Add null guard to PaymentProcessor authorise method`
- `PROJ-1234: Fix COMPILATION — missing import in ChaseMapper`

## Commit Discipline

- Each commit represents **one logical change** — no unrelated changes in a single commit
- Commit messages: present-tense imperative (`Add`, `Fix`, `Update`, not `Added`, `Fixed`)
- Always run `mvn clean compile -q` before committing
- Always run `mvn test -q` before committing if tests are affected by the change
- No secrets, credentials, or PII in any commit — if accidentally committed: rotate immediately

## Pull Request

- PR title format: `<STORY-ID>: <Story Title>`
- PR body: use `pr-description-<story-id>.md` template (generated by Code Development Agent)
- PR size: **≤ 400 lines changed** — split larger changes into stacked PRs
- Always link the PR to the Jira story via MCP

## Branch Strategy

- Never commit directly to `main` / `master`
- Feature branches cut from `develop` or `main` (as configured in `workflow-config.json`)
- Merge strategy: **squash** (default) — preserves clean history on main branch
- Delete feature branch after successful merge

## Pre-Push Checklist

- [ ] Copyright header present on all new `.java` files
- [ ] `mvn clean compile -q` succeeds
- [ ] `mvn test -q` succeeds (zero failures)
- [ ] No secrets or PII in changed files
- [ ] Commit messages follow `<TICKET-ID>: <Description>` format

## Protecting History

- **Never** `git push --force` on shared branches
- **Never** `git reset --hard` without user confirmation
- **Never** rewrite published commit history without explicit user consent
- If a secret is accidentally committed: rotate it immediately; rewrite history with `git filter-repo`
