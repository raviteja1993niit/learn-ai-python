📋 Overview

GitHub Copilot exposes an OpenAI-compatible REST API at https://api.githubcopilot.com. Any GitHub user with an active Copilot subscription can call this API directly using a standard GitHub OAuth token — no special PAT, no extra scopes, no separate API key.

✅ Prerequisites

1. GitHub Copilot Subscription

You need an active Copilot subscription. Any of these qualify:

PlanWhoModels Available
Copilot IndividualPersonal GitHub accountGPT-4o, Claude Sonnet 4, limited set
Copilot BusinessOrganization membersFull model set including Claude Opus, GPT-5.x
Copilot EnterpriseEnterprise org membersFull model set + admin-controlled access per model
⚠️ Enterprise model gating In Copilot Enterprise, each model must be explicitly enabled by an organization admin at Settings → Copilot → Policies. Even if a model exists in the API, it will return 403 if not enabled for your org.

2. GitHub CLI (gh)

The GitHub CLI is the easiest way to get a token. Install it once and authenticate:

  • Install gh CLI
    # Windows (winget)
winget install GitHub.cli

# Windows (choco)
choco install gh

# macOS
brew install gh

# Linux (apt)
sudo apt install gh
  • Authenticate
    gh auth login
# → Choose: GitHub.com → HTTPS → Login with a web browser
  • Verify
    gh auth status
# Output: ✓ Logged in to github.com account <username>
#         Token: gho_****  (scopes: repo, read:org, gist, workflow)

gh auth token
# Output: gho_XmrBf...  ← this is your Copilot API token

3. No Special Scopes Required

This is important: the GitHub Copilot API authenticates via subscription entitlement, not token scopes. The default gho_* OAuth token from gh auth login has everything needed.

🔑 Token Acquisition

Option A — gh CLI (Recommended)

# One-liner to capture token into a variable
$token = gh auth token

# Or store in environment variable
$env:GITHUB_TOKEN = gh auth token
PowerShell

Option B — Classic PAT (no scopes needed)

  1. Go to github.com → Settings → Developer Settings → Personal access tokens → Tokens (classic)
  2. Click Generate new token (classic)
  3. Give it a name, set an expiry. Do not check any scopes — leave all boxes unchecked
  4. Copy the ghp_* token and store it securely
💡 Token auto-detection in Python The following priority order is recommended in any Python app: 
import os, subprocess

def resolve_token() -> str:
    # 1. Explicit argument or environment variable
    if token := os.environ.get("GITHUB_TOKEN", ""):
        return token
    # 2. gh CLI auto-detection
    result = subprocess.run(["gh", "auth", "token"], capture_output=True, text=True, timeout=5)
    return result.stdout.strip() if result.returncode == 0 else ""

💻 PowerShell Access

PowerShell uses WinHTTP (Windows native HTTP stack) which automatically uses the Windows system certificate store and proxy settings — making it the most reliable option in corporate environments.

Basic Chat Completion

$token = gh auth token

$headers = @{
    "Authorization" = "Bearer $token"
    "Content-Type"  = "application/json"
}

$body = @{
    model    = "claude-haiku-4.5"
    messages = @(
        @{ role = "system"; content = "You are a helpful assistant." }
        @{ role = "user";   content = "What is BM25 retrieval?" }
    )
    temperature = 0.2
} | ConvertTo-Json -Depth 5

$response = Invoke-RestMethod `
    -Uri    "https://api.githubcopilot.com/chat/completions" `
    -Method POST `
    -Headers $headers `
    -Body   $body

$response.choices[0].message.content

Reusable Function

function Invoke-CopilotChat {
    param(
        [string]$Prompt,
        [string]$Model       = "gpt-4o-mini",
        [string]$SystemPrompt = "You are a helpful assistant.",
        [double]$Temperature  = 0.2
    )
    $token = gh auth token
    $headers = @{
        "Authorization" = "Bearer $token"
        "Content-Type"  = "application/json"
    }
    $body = @{
        model       = $Model
        temperature = $Temperature
        messages    = @(
            @{ role = "system"; content = $SystemPrompt }
            @{ role = "user";   content = $Prompt }
        )
    } | ConvertTo-Json -Depth 5

    $r = Invoke-RestMethod -Uri "https://api.githubcopilot.com/chat/completions" `
                          -Method POST -Headers $headers -Body $body
    return $r.choices[0].message.content
}

# Usage
Invoke-CopilotChat -Prompt "Explain LangChain in 2 sentences" -Model "claude-sonnet-4.6"

⚠️ The Corporate Proxy Problem

In enterprise environments with SSL inspection (MITM proxies), the Python openai SDK fails intermittently with a 403 PermissionDeniedError or SSL errors — even though the token is valid and PowerShell succeeds consistently.

Root Cause Analysis

🟢 PowerShell (WinHTTP) — Reliable

  • Uses Windows system certificate store
  • Trusts corporate CA cert automatically
  • Handles NTLM/Kerberos proxy auth natively
  • Connection pooling matches proxy expectations
  • ~95% success rate in tests

🔴 Python httpx (OpenAI SDK) — Unreliable

  • Uses certifi bundle (no corporate CA)
  • SSL cert verify fails → proxy rejects with 403
  • Extra x-stainless-* headers may trigger proxy inspection
  • Connection keep-alive interacts poorly with proxy state
  • ~40–60% success rate without fixes

What the Errors Look Like

# SSL certificate error
httpx.ConnectError: [SSL: CERTIFICATE_VERIFY_FAILED]
  certificate verify failed: self-signed certificate in certificate chain

# Proxy blocking the connection
openai.PermissionDeniedError: Access to this endpoint is forbidden.
  Please review our Terms of Service.

# This is the proxy rejecting Python's TLS handshake,
# NOT GitHub's API rejecting your token!
Python traceback

How to Confirm It's a Proxy Issue

# If this works in PowerShell but not Python → it's the proxy
$token = gh auth token
$headers = @{ "Authorization" = "Bearer $token"; "Content-Type" = "application/json" }
$body    = '{"model":"gpt-4o-mini","messages":[{"role":"user","content":"hi"}],"max_tokens":5}'

🔧 Workarounds & Fixes

Fix 1 — PowerShell Subprocess (Most Reliable on Windows)

Bypass Python's HTTP stack entirely. Call PowerShell's Invoke-RestMethod from Python via subprocess. WinHTTP handles all proxy and SSL negotiation natively.

import json, subprocess

def copilot_chat_via_powershell(token, model, messages, temperature=0.2):
    """Uses WinHTTP via PowerShell subprocess — most reliable behind corporate proxy."""
    body = json.dumps({
        "model": model,
        "messages": messages,
        "temperature": temperature,
    }).replace("'", "\'")

    ps_script = f"""
$headers = @{{
    "Authorization" = "Bearer {token}"
    "Content-Type"  = "application/json"
}}
$body = '{body}'
$r = Invoke-RestMethod -Uri "https://api.githubcopilot.com/chat/completions" `
                       -Method POST -Headers $headers -Body $body
$r.choices[0].message.content
"""
    result = subprocess.run(
        ["powershell", "-NoProfile", "-Command", ps_script],
        capture_output=True, text=True, timeout=60,
    )
    if result.returncode == 0 and result.stdout.strip():
        return result.stdout.strip()
    raise RuntimeError(result.stderr.strip())
Python

Fix 2 — httpx with Retry Logic (Cross-Platform)

Use httpx directly with a fresh client per request and exponential backoff on 403s.

import httpx, json, time, os

def copilot_chat(token, model, messages, provider="GitHub Copilot",
                   max_retries=5, retry_delay=1.5):
    url = "https://api.githubcopilot.com/chat/completions"
    ssl_verify = os.environ.get("SSL_VERIFY", "true").lower() != "false"
    headers = {
        "Authorization": f"Bearer {token}",
        "Content-Type":  "application/json",
        "Accept":         "application/json",
        "Editor-Version": "vscode/1.95.0",   # helps some proxies allow it
    }
    body = json.dumps({"model": model, "messages": messages})

    for attempt in range(1, max_retries + 1):
        # New client per attempt — avoids keep-alive proxy state issues
        with httpx.Client(verify=ssl_verify, timeout=60.0) as client:
            r = client.post(url, headers=headers, content=body)
        if r.status_code == 200:
            return r.json()["choices"][0]["message"]["content"].strip()
        if r.status_code in (403, 429, 503) and attempt < max_retries:
            time.sleep(retry_delay * attempt)
            continue
        r.raise_for_status()
    raise RuntimeError(f"Failed after {max_retries} attempts")
Python

RAG Chain with GitHub Copilot LLM

from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough

prompt = ChatPromptTemplate.from_template("""
Answer the question based only on the following context:
{context}

Question: {question}
""")

chain = (
    {"context": RunnablePassthrough(), "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

result = chain.invoke({
    "context":  "BM25 is a ranking function based on term frequency.",
    "question": "What is BM25?",
})

⚠️ Corporate proxy note for LangChain LangChain's ChatOpenAI uses the OpenAI SDK internally, which uses httpx. The same SSL inspection issues apply. Use truststore.inject_into_ssl() at app startup, or pass a custom http_client: 
import httpx
http_client = httpx.Client(verify=False)  # or verify="path/to/corp-ca.pem"
llm = ChatOpenAI(..., http_client=http_client)

Multi-step RAG Workflow Graph

from langgraph.graph import StateGraph, END
from typing import TypedDict, List

class RAGState(TypedDict):
    question: str
    chunks:   List[str]
    answer:   str

def retrieve(state: RAGState) -> RAGState:
    # BM25 retrieval step
    state["chunks"] = bm25_retriever.retrieve(state["question"])
    return state

def generate(state: RAGState) -> RAGState:
    context = "\n\n".join(state["chunks"])
    response = llm.invoke(f"Context:\n{context}\n\nQ: {state['question']}")
    state["answer"] = response.content
    return state

graph = StateGraph(RAGState)
graph.add_node("retrieve", retrieve)
graph.add_node("generate", generate)
graph.set_entry_point("retrieve")
graph.add_edge("retrieve", "generate")
graph.add_edge("generate", END)

rag_app = graph.compile()
result  = rag_app.invoke({"question": "What is BM25?", "chunks": [], "answer": ""})
print(result["answer"])

🤝 CrewAI Integration

CrewAI supports custom LLM endpoints via LiteLLM under the hood. You can point it to the GitHub Copilot API using openai/<model> naming convention with custom api_base.

pip install crewai langchain-openai
bash

Method A — LangChain LLM (Recommended)

import truststore; truststore.inject_into_ssl()
import subprocess
from langchain_openai import ChatOpenAI
from crewai import Agent, Task, Crew, Process

token = subprocess.run(["gh", "auth", "token"],
                       capture_output=True, text=True).stdout.strip()

llm = ChatOpenAI(
    openai_api_key=token,
    openai_api_base="https://api.githubcopilot.com",
    model="claude-sonnet-4.6",
    temperature=0.2,
)

researcher = Agent(
    role="Document Researcher",
    goal="Find and extract relevant information from the document",
    backstory="Expert at reading and analyzing technical documents",
    llm=llm,
    verbose=True,
)

writer = Agent(
    role="Answer Writer",
    goal="Write clear, accurate answers based on research",
    backstory="Expert at synthesizing information into clear answers",
    llm=llm,
    verbose=True,
)

research_task = Task(
    description="Research the document for information about {topic}",
    agent=researcher,
    expected_output="Key facts and relevant chunks about the topic",
)

writing_task = Task(
    description="Write a comprehensive answer based on the research",
    agent=writer,
    expected_output="A clear, accurate answer to the user's question",
)

crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, writing_task],
    process=Process.sequential,
)

result = crew.kickoff(inputs={"topic": "BM25 retrieval algorithms"})
print(result)
